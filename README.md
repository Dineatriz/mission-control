🚀 Mission Control Simulator is a space mission management simulator. This project focuses on constant realism and reactive feedback. I didn't use libraries, frameworks, or external resources.

The player takes control of a space mission divided into 4 levels: Launch, Orbit, Crisis, and Landing. The objective is to manage the limited resources that appear on the left (Fuel, Energy, and Oxygen) while responding to emergency events that deplete these resources.

I really like the reactive animation on the screen; the rocket's altitude, smoke trails, flame size, and lateral drift react in real time to the current thrust and gravity values.

The game has real-time audio generated through code; that is, all sound design (such as launch sequences, low resource alarms, victory fanfares, and collision sounds) is generated by code using the Web Audio API.

On the left, there's also a dynamic telemetry system that provides real-time monitoring of G-force, orbital trajectory data, speed, and ETA (Estimated Time of Arrival) for phase completion.

One of the latest additions was the consequences: Choices define internal indicators that influence future events (for example, ignoring a small leak early on can trigger a tank failure later in the game).

There's feedback in the game elements; basically, the user interface reacts to the ship's state with vibration effects when integrity is low and dynamic color changes (yellow/red) as resources become critical.

One thing I really liked adding was the conditional events: The system reads the ship's current state to trigger context-specific events (for example, low oxygen levels during the crisis phase trigger a Hypoxia event).

In the end, there is a score and ranking based on the streak of good choices and the number of options selected.

Technologies used: JavaScript (ES6+)

Rendering: HTML5 Canvas (Central Simulation) and SVG (Interface)

Audio: Web Audio API (Real-time sound synthesis)

Styling: CSS3 with dynamic variables and keyframe animations
